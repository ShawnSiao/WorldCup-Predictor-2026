# 预测与复盘方法

[English](methodology.md) | [简体中文](README.zh-CN.md) | [更新日志](../CHANGELOG.md)

## 推理模型

所有预测推理均指定使用 ChatGPT 5.5 超高推理模型。

仓库只公开简明推理摘要，不保存隐藏推理链或私有推理过程。

## 预测输入

每场赛前预测在给出结果前，必须先完成一轮覆盖检查，至少包括：

- 战术与预期战术对位
- 球员、阵容强度与预计首发质量
- 伤病、停赛与球员可用性
- 赛程、休息天数、旅行负担、开球时间与场地因素
- 交锋记录、赛事历史与相关历史模式
- 舆情、媒体叙事与球迷期待
- 天气、气候、草皮、海拔与场馆条件
- 心理压力、比赛动机、小组赛或淘汰赛目标
- 可靠来源中的赔率与赔率变动，并明确标注为市场背景
- 可靠专家、官方转播或权威编辑来源的观点

如果某项输入暂缺、过期或未验证，预测必须明确标注数据缺口，说明其对信心等级的影响，并且不得把缺失项当作已确认依据。

## 预测输出

小组赛预测包含：

- 主胜概率
- 平局概率
- 客胜概率
- 预测比分
- 信心等级
- 发布配图
- 事实依据
- 预测覆盖检查
- 推理逻辑
- 推理摘要
- 风险因素
- 平台发布文案
- 投资建议免责声明

淘汰赛预测需要补充晋级概率，并区分常规时间结果、加时赛和点球场景。

## 发布文案

每份预测应包含适合抖音、小红书、微博和微信发布的文案。

平台文案应简洁、基于事实，并明确标注为赛事预测。文案必须包含「不构成任何投资建议」的免责声明。各平台账号说明性文案统一维护在[平台发布文案](platform-copy.zh-CN.md)。

每份预测和日报都应引用 `assets/cards/` 下由 `$imagegen` 生成的真实位图配图。使用以下指令形式：

```text
$imagegen: 生成【社交平台赛事预测配图】，16:9 横版，真实位图图片，用于抖音、小红书、微博和微信分享；中文文档配图的主要比赛信息必须使用简体中文，可在画面合适位置保留英文队名/赛事信息作为辅助文字；不要生成 SVG，不要生成 HTML，不要生成代码图，不要生成线框图，不要使用官方 FIFA 标志或水印。
```

每日自动任务在验证通过后，应读取当前已连接 Gmail profile 的邮箱地址，并将当日摘要和生成配图发送到该 Google 邮箱。不得使用 Git 配置邮箱、GitHub 账号邮箱或仓库元数据作为收件人。

## 复盘方法

赛后复盘用于对比赛前预测与官方最终赛果。

每份复盘应记录：

- 实际比分和胜者
- 关键事件和转折点
- 预测胜者是否命中
- 预测比分是否接近
- 哪些判断成立
- 哪些判断失败
- 后续比赛的模型调整记录

复盘评级：

- `correct`：核心结果命中，比赛走势接近。
- `partial`：判断有参考价值，但存在明显偏差。
- `wrong`：核心结果判断错误。
